# API Documentation

This is the API documentation for the Timeloop API service, which will be used as the backend service for the Timeloop web application. This API will handle user authentication and authorization, as well as the creation, retrieval, updating, and deletion of user data. Also, it will handle the creation, retrieval, updating, and deletion of user calendars and events.

## API Endpoints

The Timeloop API service has different endpoints for different types of requests. The following are the available endpoints for the Timeloop API service:

### Auth Routes

The auth routes are used to register, log in, and log out a user. The user must be authenticated to access the other routes. The auth routes are as follows:

#### POST /api/v1/auth/signup

Registers a new user

- Request Body:
  - `username`: string
  - `first_name`: string
  - `last_name`: string
  - `email`: string
  - `password`: string
- Request Headers:
  - `api-key`: string
- Response:
  - `status`: string
  - `message`: string
  - `data`: object

#### POST /api/v1/auth/login

- Logs in a user
- Request Body:
  - `email`: string
  - `password`: string
- Request Headers:
  - `api-key`: string
- Response:
  - `status`: string
  - `message`: string
  - `data`: object

#### GET /api/v1/auth/logout

- Logs out a user
- Response:
  - `status`: string
  - `message`: string
  - `data`: object

## API Key

The API key is a unique string that is used to authenticate requests to the Timeloop API service. It is required to be included in the headers of all requests to the API service. The API key is generated by the Timeloop API service and is unique to each user. It is used to ensure that only authorized users can access the API service.

## Error Handling

The Timeloop API service will return appropriate HTTP status codes and error messages for each request. The following are the possible HTTP status codes and their corresponding error messages:

- `200 OK`: The request was successful and the response contains the requested data.
- `201 Created`: The request was successful and a new resource was created.
- `400 Bad Request`: The request was invalid or could not be processed.
- `401 Unauthorized`: The request was not authorized and the user needs to log in.
- `403 Forbidden`: The request was not allowed and the user does not have permission to access the resource.
- `404 Not Found`: The requested resource was not found.
- `500 Internal Server Error`: An unexpected error occurred on the server.
